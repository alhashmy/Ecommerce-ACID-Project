# 🛍️ Full E-commerce & Inventory Management System (ACID Compliant)

منصة تجارة إلكترونية متكاملة ونظام مصغر لإدارة المستودعات والعمليات اللوجستية، مصمم ومبني لإثبات وتطبيق المعايير الهندسية الصارمة لقواعد البيانات والمعروفة بـ **ACID Compliance** حمايةً للمخازن والبيانات المالية من التداخل أو الأخطاء الحسابية بلحظات الضغط العالي والطلب المتزامن.

---

## 🚀 المميزات الرئيسية للمشروع (Key Features)

* **إدارة المخازن الآمنة (ACID Transactions):** تطبيق نظام الـ `COMMIT` والـ `ROLLBACK` البرمجي المقترن بقيد قاعدة البيانات `CHECK (stock_quantity >= 0)`. في حال طلب الزبون كمية أكبر من المتوفر بالمخزن، يرفض النظام العملية استباقياً (Early Validation) ويقوم بعمل تراجع مادي كامل لمنع حدوث أي تداخل مالي أو مخزني خبيث.
* **نظام التحديث اللحظي (Live Polling System):** واجهة انتظار تفاعلية للزبون مبنية بالـ JavaScript Fetch API، تقوم بفحص السيرفر كل ثانيتين لتقلب شاشة الزبون تلقائياً فور اتخاذ الأدمن قرار القبول أو الرفض بالخلفية وبدون الحاجة لتحديث الصفحة يدوياً.
* **شاشة تتبع الشحن والفواتير للزبائن (`/track`):** تتيح للزبون إدخال رقم الطلب (Order ID) الخاص به لاستخراج تفاصيل الفاتورة، السعر النهائي، والتحقق من مسار الشحن المباشر عبر شريط تتبع حركي متطور.
* **لوحة تحكم إدارية متكاملة (ERP Dashboard):** لوحة تحكم محمية بنظام الجلسات (`/login`) تتيح لمدير العمليات التحكم الكامل بالمخازن (إضافة منتج جديد، تحديث الأسعار والكميات والصور، وإضافة خصومات خاصة بكل منتج)، بالإضافة إلى إدارة مسار الشحن والتوصيل (Pending ➡️ Approved ➡️ In Transit ➡️ Delivered).
* **محرك الخصومات الذكي (Promo Code Engine):** يدعم إدخال وتطبيق أكواد الخصم المتقدمة استباقياً مع معالجة تعارض الأنواع البرمجية ببيئات بايثون وفصل الحسابات الكسرية (مثل كود `ASHUR` الذي يمنح خصماً فورياً بقيمة 50% على إجمالي الفاتورة).
* **مظهر عصري متجاوب (Responsive UI & Dark Mode):** واجهات بروفيشينال تدعم الوضع الداكن بالكامل ومبرمجة لحفظ خيار المستخدم داخل الـ LocalStorage الخاص بالمتصفح.

## 🛠️ التكنولوجيا المستخدمة (Tech Stack)

* **Backend:** Python 3.13, Flask Framework
* **Database:** PostgreSQL (psycopg2)
* **Frontend:** HTML5, CSS3, JavaScript (ES6, Live Polling)

---

## 🗄️ هيكل قاعدة البيانات (Database Architecture)

المشروع يعتمد على جدولين مترابطين بعلاقة (One-to-Many Relationship):
1. **جدول المنتجات (`products`):** يحتوي على تفاصيل البضائع، الأقسام الفئوية (Laptops, Phones, Accessories)، الكميات المتوفرة، ونسبة خصم المنتج.
2. **جدول الطلبات (`orders`):** يحتوي على بيانات الزبائن، العناوين، المنتج المرتبط عبر (Foreign Key)، الحساب الكلي، الكود المستخدم، وحالات التوصيل الحركية.
